Windows平台调试Linux程序

0、blah blah blah……

相对于Windows平台上的IDE工具，Linux在这方面稍差，命令行的调试始终没有GUI的调试直观和简单，也可能是没有习惯使用Linux的调试工具。

其实，还是有办法可以在Windows平台使用IDE来调试Linux程序的。

1、准备工作

  • Windows 7
  • Visual Studio 2015 Community（简称vs2015）
  • VirualGDB 5.2r7
  • VMware Workstation 12 Pro
  • VMware中安装Linux，如CentOS、Ubuntu

1、创建项目

  • 打开vs2015
  • 创建项目，如下图所示

  • 配置与Linux环境的连接（Vmware环境），根据需要进行选择
  • 配置SSH连接。示例环境已经配置过SSH，可以直接选择，第一次使用VisualGDB，会要求配置新的SSH

  • 一路next……，会要求配置每次编译的时候由Windows上传至Linux的文件类型，配置完成后点finish
  • 一个适用于Linux环境的项目已经创建好了，但还有其他的内容需要配置。点击 VisualGDB Startup Project Properties 进行VisualGDB参数配置

  • Projcet settings，默认即可
  • Unit Test，默认即可
  • MSBuild settings，需要重点配置command build settings，

    其中，
  1. Processor macros，宏定义，项目中使用的宏可以在此填写
  2. Include directories，编译时引用文件的目录
  3. Libary directories，编译时库文件的目录
  4. Libary names，使用的库文件，需要注意的是库文件的名字必须是libxxxx.so或者libxxxx.a样式的，如果只是xxxx.so貌似是错误
  • Debug settings，在此处配置需要调试的可执行文件。在Debug -> Debugged executable中
  • 其余配置项默认即可（作者目前没有使用到）
  • 编译，通过SSH可以查看在Linux环境中的编辑结果如何，
  • 至此，已经可以在vs2015中像调试Windows程序那样调试Linux程序了，断点、分步调试都是熟悉的样子
